## Brand & Style

This design system is built for the focused engineer. It adopts a **Developer-Centric / Minimalist** aesthetic, heavily inspired by modern IDE environments. The goal is to reduce cognitive load by using a dark, low-contrast UI that recedes into the background, allowing the code and logic to remain the primary focus. 

The visual identity is defined by a "Sidebar-First" architecture, using an **Activity Bar** for high-level navigation and a nested **Sidebar** for contextual information. The emotional response should be one of "Flow State"—professional, precise, and utilitarian. It avoids decorative flourishes in favor of structural clarity, thin borders, and monospaced accents.

## Colors

The palette is anchored in **Deep Charcoal and Slate**, utilizing a multi-layered dark mode strategy. Unlike standard web dark modes, this system uses subtle shifts in value to define depth rather than shadows.

- **Backgrounds:** The darkest shade (`#1E1E1E`) is reserved for the editor area. The Activity Bar uses a slightly warmer grey (`#333333`) to distinguish the primary navigation anchor.
- **Difficulty Accents:** High-contrast semantic colors are used exclusively for problem difficulty: **Emerald** for Easy, **Amber** for Medium, and **Rose** for Hard. These should be used sparingly as text labels or small indicators.
- **Primary Action:** The "VS Code Blue" (`#007ACC`) is used for active states, primary buttons, and focus indicators.
- **Syntax Highlighting:** Follows a Monokai-inspired logic: vibrant pinks, yellows, and greens specifically within the code editor container.

## Typography

This design system uses a dual-font strategy to balance UI readability with technical precision. 

1. **UI Sans:** `Hanken Grotesk` provides a sharp, contemporary feel for all interface elements, menus, and descriptive text. It is clean and highly legible at small sizes.
2. **Technical Mono:** `JetBrains Mono` is used for the code editor, terminal, and metadata labels (like difficulty levels and runtime stats).

**Hierarchy:**
- Headings are kept compact; we prioritize density over large display type.
- Labels are often uppercase with slight tracking to mimic the "Status Bar" aesthetic of developer tools.
- Code blocks must utilize a 13px or 14px size for optimal legibility during long sessions.

## Layout & Spacing

The layout follows a **Fixed-Pane Grid** model. Rather than fluid margins, the UI is divided into rigid functional zones separated by 1px borders (the "gutter").

- **Activity Bar:** Fixed at 48px on the extreme left.
- **Sidebar:** A collapsible pane for file explorers or problem lists.
- **Editor Area:** The main flexible region that occupies the remaining space.
- **Bottom Panel:** A resizable area for Terminal and Test Results.

**Density:** The design system is "High Density." Spacing is tight (increments of 4px) to ensure as much information as possible is visible without scrolling.

## Elevation & Depth

Depth is communicated through **Tonal Tiering** and **1px Borders** rather than shadows. In a dark IDE environment, shadows can appear muddy or disappear entirely.

- **Level 0 (Bottom):** Editor surface (`#1E1E1E`).
- **Level 1 (Sidebars):** Navigation and panels (`#252526`).
- **Level 2 (Active/Floating):** Context menus and tooltips. These use a slightly lighter grey with a subtle 1px border (`#454545`) to separate them from the background.
- **Focus States:** Active inputs or editor lines are highlighted with a primary blue border or a subtle background tint, never a heavy drop shadow.

## Shapes

To maintain the technical, "engineered" look of a code editor, the design system utilizes **Sharp (0px)** corners for almost all structural elements.

- **Panels & Tabs:** 0px radius.
- **Buttons:** 2px radius (near-sharp) only to provide a tiny hint of "tappability."
- **Inputs:** 0px radius with a 1px solid border.
- **Chips/Badges:** 0px radius, using Monospaced text to reinforce the technical aesthetic.

## Components

### Buttons
- **Primary:** Solid blue (`#007ACC`) with white text. Square corners.
- **Secondary:** Ghost style with a 1px grey border.
- **Icon Buttons:** Transparent background, primary blue icon on hover.

### Tabs (The "Editor Tab")
Tabs should feel like physical files. The active tab has a top border highlight (2px blue) and a background matching the editor. Inactive tabs are darker with no highlight.

### Problem Lists
Items should be treated as "rows." On hover, the entire row receives a subtle background highlight (`#2A2D2E`). Difficulty indicators (Easy/Med/Hard) appear as colored labels on the right side of the row.

### Inputs & Editor
- **Input Fields:** Darker than the background, 1px border that turns blue on focus.
- **The Editor:** Line numbers on the left, "Gutter" for breakpoints or markers, and a vertical "Ruler" at 80 or 120 characters.

### Activity Bar Icons
Icons should be thin-line (24px), centered within the 48px bar. The active icon should have a 2px vertical blue line on its left edge.
